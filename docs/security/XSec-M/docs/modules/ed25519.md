# Ed25519 Signature Verification

The Ed25519 module provides high-performance asymmetric signature verification. Unlike standard RSA, Ed25519 uses Twisted Edwards curves, offering smaller signatures and faster computation with superior security.

XyPriss Security implements a simplified, Go-backed Ed25519 primitive that removes the complex boilerplate of DER encoding and SPKI conversion typically required in Node.js.

## Methods

### `Cipher.crypto.ed25519Verify(publicKey, data, signature)`

Verifies a detached Ed25519 signature.

**Parameters:**

- `publicKey`: `string | Uint8Array` - The 32-byte raw public key (Hex string or byte array).
- `data`: `string | Uint8Array` - The original data that was signed.
- `signature`: `string | Uint8Array` - The signature to verify (Base64 string or byte array).

**Returns:** `boolean` - `true` if the signature is valid, `false` otherwise.

## High-Performance Large Buffer Support

XyPriss Security automatically optimizes communication between the Go core and Node.js. For large data payloads (e.g., verifying multi-megabyte audit files), the framework transparently switches from CLI arguments to high-speed `stdin` piping.

This mechanism:

1. **Bypasses OS Limits**: Successfully handles data that exceeds the `E2BIG` (Argument list too long) threshold of approximately 128KB.
2. **Maintains Performance**: Uses low-overhead process standard streams for near-instant data transfer.

## Example

```typescript
import { Cipher } from "xypriss-security";

const authorPubKey =
  "7dc962b27af59bc7a9e0d0d2c155989e0ec5a80fed78ac1375ae95038a31afd5";
const data = "System Integrity Verified - G3 Protocol Active";
const signature =
  "gcV+Lm7C2y6sn0H3KH9yC741Al968YzuqvGkyp5G4oybekBeFavs0/LypaAGZrt7qLDTOL86nfRs5qE9DT8sAA==";

const isValid = Cipher.crypto.ed25519Verify(authorPubKey, data, signature);

if (isValid) {
  console.log("✅ Identity and data integrity confirmed.");
}
```

## Security Note

This implementation uses the Go standard library's `crypto/ed25519` package, which is constant-time and resistant to timing attacks. By using our lib's native bridge, you avoid the complex and error-prone `crypto.createPublicKey` DER-wrapper logic.
